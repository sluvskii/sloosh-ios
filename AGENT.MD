Привет! Мы продолжаем разработку iOS-приложения для просмотра фильмов и сериалов "sloosh" (название всегда пишется с маленькой буквы). Разработка ведется на Windows без использования Mac, поэтому сборка `.ipa` настроена через GitHub Actions, а установка на iPhone происходит через Sideloadly.

**Что уже сделано в проекте:**
1. Настроена базовая архитектура на SwiftUI (iOS 18.0 target).
2. Настроены CI/CD пайплайны в `.github/workflows/ios-build.yml`, которые успешно собирают и отдают unsigned `.ipa` файл из `.xcarchive`.
3. Реализована навигация через `TabView` с тремя вкладками: "Главная" (`HomeView`), "Поиск" (`SearchView`) и "Профиль" (`ProfileView`).
4. Внедрена поддержка системной светлой/темной темы (`UIColor.systemBackground` и семантические цвета `.primary`/`.secondary`).
5. Установлен фирменный акцентный цвет: кислотно-желто-зеленый (`#B3FF00`).
6. Добавлена векторная иконка приложения и логотип в `Assets.xcassets`.
7. Созданы базовые модели данных (`Movie`) и замоканы списки фильмов/сериалов в `MockData.swift`.
8. Сверстаны переиспользуемая карточка фильма (`MovieCardView`) и экран деталей (`MovieDetailView`).

**Наша следующая задача:**
Нужно заменить статические заглушки (`MockData`) на реальные данные из сети.
1. Напиши сетевой слой (Network Service) для получения данных о фильмах. Давай использовать бесплатное API (например, TMDB — The Movie Database).
2. Обнови модель `Movie` под формат JSON-ответов выбранного API (добавь поддержку `Codable`).
3. Реализуй загрузку и кэширование реальных постеров фильмов в `MovieCardView` и `MovieDetailView` (с использованием `AsyncImage` или стороннего легковесного решения).
4. Обнови `HomeView`, чтобы карусели "В тренде" и "Новые сериалы" заполнялись реальными данными из сети асинхронно.

Пожалуйста, изучи текущую структуру проекта (особенно `Movie.swift`, `HomeView.swift` и `MovieCardView.swift`) и предложи код для сетевого слоя.
