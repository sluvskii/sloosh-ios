Привет! В предыдущих чатах мы успешно связали iOS-приложение `sloosh` с бэкендом NeoMovies API и интегрировали видеоплеер внутрь приложения, чтобы отказаться от переадресации в Safari.

**Что уже было сделано по плееру:**
1. Мы пытались реализовать нативный `AVPlayer` и написать новый эндпоинт для API, который возвращал бы чистые HLS-ссылки (`.m3u8`).
2. Однако выяснилось, что бэкенд на проде не обновляется (доступа к его коду нет), поэтому **мы отказались от кастомного JSON-эндпоинта**.
3. **Финальное и рабочее решение:** мы используем тот же подход, что и в веб-версии приложения (`neomovies-web-main`). Бэкенд уже умеет отдавать готовую HTML-страницу с плеером балансера по адресу `/api/v1/players/cdn/kp/{kp_id}`.
4. В файле `NeoMoviesService.swift` метод `fetchStreamURL` теперь просто генерирует этот URL (предварительно очищая префикс `kp_` из ID).
5. Экран `PlayerSelectionView.swift` был полностью переписан:
   - Вся сложная логика с парсингом сезонов и серий удалена.
   - Внедрена структура `WebView` (обёртка над `WKWebView` из `WebKit`).
   - Теперь по нажатию на кнопку "Смотреть" открывается модальное окно, внутри которого загружается веб-плеер балансера. Плеер настроен на `allowsInlineMediaPlayback = true`, чтобы корректно отображаться в приложении.

**Текущее состояние проекта:**
Приложение успешно собирается, ошибка `1011 (NSURLErrorBadServerResponse)` устранена (так как мы больше не обращаемся к несуществующему эндпоинту), и видео должно воспроизводиться в интегрированном веб-плеере.

**Важные нюансы проекта:**
- Название приложения пишется с маленькой буквы: **sloosh**.
- Используем системные цвета `.primary`, `.secondary` и наш `SlooshTheme.accent` (кислотно-зеленый).
- Разработка ведется на Windows без Mac (терминальные команды для git/xcodebuild блокируются песочницей, `sandbox config not found`). Все коммиты делаем через UI редактора, а сборку проверяем через GitHub Actions (собирается `.ipa`).
- **Бэкенд-API не подлежит изменению!** Мы используем только те эндпоинты, которые уже существуют на `https://api.neomovies.ru/api/v1` (ориентируйся на веб-версию `neomovies-web-main`, чтобы понимать, как работать с API).
