Привет! Мы продолжаем разработку iOS-приложения для просмотра фильмов и сериалов "sloosh" (название всегда пишется с маленькой буквы). Разработка ведется на Windows без использования Mac, поэтому сборка `.ipa` настроена через GitHub Actions, а установка на iPhone происходит через Sideloadly.

**Что уже сделано в проекте:**
1. Настроена базовая архитектура на SwiftUI (iOS 18.0 target).
2. Настроены CI/CD пайплайны, которые успешно собирают и отдают unsigned `.ipa` файл из `.xcarchive`.
3. Реализована навигация через `TabView` (Главная, Поиск, Профиль).
4. Установлен фирменный акцентный цвет: кислотно-желто-зеленый (`#B3FF00`), добавлена векторная иконка и логотип.
5. Настроен сетевой слой (`NeoMoviesService`) и экраны (Home, MovieDetail) получают реальные данные.
6. Внедрен нативный плеер на базе `AVPlayer` (`PlayerView.swift`).

**Текущая проблема (Ошибки плеера 1003, 1011 и парсинг сериалов):**
Мы попытались использовать CDN балансер `api.rstprgapipt.com`, доставая прямые `.m3u8` ссылки и скармливая их `AVURLAsset` с заголовками. 
Однако `AVPlayer` все равно выбрасывает ошибки:
- `NSURLErrorDomain error - 1003` (Не удается найти хост)
- `NSURLErrorDomain error - 1011` (403 Forbidden / Bad Server Response)
- При загрузке сериалов возникает ошибка декодирования данных JSON: "The data couldn't be read because it is missing".
Причина: Сервера защищены от прямого доступа, требуют специфичных динамических токенов/куки и проверки заголовков, которые тяжело поддерживать обычными HTTP-запросами.

**ЗАДАЧА ДЛЯ СЛЕДУЮЩЕГО АГЕНТА:**
Отказываемся от текущей реализации `PlayerService` (CDN `api.rstprgapipt.com`). 
Нужно реализовать балансер **Alloha**, полностью скопировав логику из рабочего Android-приложения (`neomovies-android-main`).

**Детальный план действий:**
1. **Изучить исходники Android-версии:**
   - Путь: `c:\Users\1\AndroidStudioProjects\sloosh-ios\SlooshIOS\neomovies-android-main\app\src\main\java\com\neo\neomovies\alloha`
   - Особое внимание на файлы:
     - `AllohaParser.kt`: Парсит страницу в невидимом `WebView`, перехватывает сетевые запросы, инжектит JS, собирает нужные динамические заголовки (`accepts-controls`, `authorizations`) и ловит ссылку на `master.m3u8`.
     - `HlsProxyServer.kt`: Локальный HTTP-прокси, который проксирует `.ts` сегменты, подставляя собранные заголовки, чтобы обойти 403 Forbidden от серверов Alloha.
     - `AllohaRepository.kt`: Запрашивает начальную `iframe` ссылку через API `https://api.alloha.tv/?token=...&kp=...`.

2. **Реализовать аналог `AllohaParser` на iOS:**
   - Использовать невидимый `WKWebView` (SwiftUI / UIKit).
   - Загрузить туда `iframe` плеера Alloha.
   - Использовать механизмы `WKUserScript` для инъекции JS-кода, аналогичного тому, что в `buildWrapperHtml` (Android), чтобы перехватить `XMLHttpRequest` / `fetch` / `WebSocket` и вытащить ссылку на `.m3u8` вместе со словарем заголовков.
   - Наладить мост между JS и Swift (через `WKScriptMessageHandler`).

3. **Обход защиты при воспроизведении (AVPlayer):**
   - Если `AVURLAsset(url: options: ["AVURLAssetHTTPHeaderFieldsKey": headers])` будет недостаточно для `.ts` сегментов, потребуется написать аналог `HlsProxyServer` (можно использовать `AVAssetResourceLoaderDelegate` для перехвата запросов к `.m3u8` и `.ts` и подмены заголовков на лету, это нативный и более легкий путь на iOS вместо поднятия локального web-сервера).

   - Убедиться, что сериалы работают корректно (парсинг сезонов и эпизодов через `AllohaRepository.kt` эквивалент).
